Controllers

  • Controllers are defined as a constructor function
  • Whenever a ngController directive is found in the DOM, a new controller is instantiated
  • A new child scope is created, and injected into the Controller's constructor
  • The controller is responsible for setting up the initial state of the $scope object
  • The controller can also add behavior to the $scope, by adding functions to the object
  • Controllers and Directives both have a reference to a $scope, but not to each other.
    • This makes the controllers view agnostic, enforcing the separation of concerns, and improving testability
  • A controller shouldn't try to do too much. It should only contain the business logic needed for a single view
  • A controller should not do any kind of DOM manipulation. (That's what directives are for)
