How does it work?

  • Most of the magic is in the DOM compiler
    • Angular is bootstrapped just after the browser loads the DOM
    • The entire DOM is parsed, looking for Angular syntax
    • The DOM is modified in place (as necessary) to attach behavior
  • Angular gets most of its power as an extension of the DOM
    • Angular consumes the DOM, not string templates
    • This two way linking results in a "live view"
    • The binding is transparent. The developer does not need to make any special calls to refresh the view
    • Other templating frameworks try to work around the DOM, with less impressive results.
      • Consume a static string template
      • Combine it with data to produce a new string
      • Inject the new string back into an element using innerHTML
      • Any changes to the data model require the template to be re-processed
