What about jQuery?

  • You don't really need it any more
  • Angular ships with jqLite
    • This is a stripped down version of jQuery
    • Only the essentials (DOM selectors and manipulators mostly)
    • Used by the internals of Angular
    • Exposed as angular.element()
  • If you include jQuery first, Angular will use it instead of jqLite
  • Angular completely restructures the way you write applications.
    • If you find yourself using jQuery to search the DOM to find some element, you are probably doing something wrong (the hard way).
    • Use a directive and/or some data binding instead.
