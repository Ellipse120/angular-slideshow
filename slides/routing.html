Routing

Routing is the key to creating a robust, single page application

  • Routing allows your application to use the URL to dictate part of the application's state
  • Based on the URL path, you can change the view state of the application
  • This is accomplished without any page reloads, by using the part of the URL after #, combined with the browser history api
  • Provides the added benefit of deep linking to a specific view state in the application via a URL
  • Angular provides the ngRoute module
    • This provides basic routing, but does not support nested views
  • I prefer the 3rd party AngularUI Router
    • Has a more sophisticated, state based model, with arbitrary nesting
